# Silva Pinto Advocacia — Painel de Oportunidades Sistema interno de inteligência: coleta automática duas vezes ao dia (09:00 e 17:00) de: 1. **Concursos públicos abertos** para inscrição em todo o Brasil 2. **Concursos em fase de recursos/anulação** (sua área de atuação direta) 3. **Decisões judiciais (jurisprudência)** sobre concursos públicos ## Arquitetura - Flask + Anthropic (Claude Haiku 4.5 com web_search) - SQLite local para armazenamento - Cron externo (cron-job.org) dispara o endpoint `/cron/diaria` - Interface web protegida por senha - Mesma identidade visual do Gerador de Petições ## Deploy no Render ### 1. Criar novo repositório no GitHub Suba todos esses arquivos para um novo repositório (separado do gerador de petições). ### 2. Criar serviço no Render - Vá em "New" → "Web Service" - Conecte o repositório - Plano: **Starter** ($7/mês) - Build Command: `pip install -r requirements.txt` - Start Command: `gunicorn app:app --worker-class gthread --workers 1 --threads 4 --timeout 600 --graceful-timeout 600 --bind 0.0.0.0:$PORT` ### 3. Configurar Environment Variables No painel do serviço, aba "Environment", adicione: | Variável | Valor | |----------|-------| | `ANTHROPIC_API_KEY` | Sua chave da API Anthropic | | `ACCESS_TOKEN` | Senha que você usará para acessar o painel | | `CRON_SECRET` | Senha aleatória — usada apenas pelo cron externo | | `MODEL_NAME` | `claude-haiku-4-5-20251001` (já é o default) | ### 4. Configurar cron externo Vá em **https://cron-job.org** (grátis) e crie 2 jobs: **Job 1 — coleta da manhã:** - URL: `https://SEU-SERVICO.onrender.com/cron/diaria?secret=SUA_CRON_SECRET` - Schedule: `09:00` todos os dias (UTC-3 — atenção: cron-job.org usa UTC, configure 12:00 UTC para 09:00 Brasília) - Method: GET **Job 2 — coleta da tarde:** - URL: igual à acima - Schedule: `17:00` Brasília (= 20:00 UTC) - Method: GET ### 5. Acessar `https://SEU-SERVICO.onrender.com/` Login com a senha definida em `ACCESS_TOKEN`. ## Custos estimados - Render Starter: $7/mês - API Claude Haiku (2x/dia × 30 dias × ~$0.05 cada): ~$3/mês - Cron-job.org: grátis **Total: ~$10/mês** ## Endpoints - `GET /` — dashboard (precisa login) - `GET /login` — tela de login - `POST /cron/manual` — disparo manual (precisa login) - `GET/POST /cron/diaria?secret=XXX` — disparo do cron externo - `GET /api/oportunidades` — lista as oportunidades - `POST /api/oportunidades//marcar_lido` - `POST /api/oportunidades//arquivar` - `GET /api/status` — estatísticas - `GET /health` — health check ## Banco de dados SQLite em `/tmp/oportunidades.db`. **Atenção:** o `/tmp` é apagado quando o Render reinicia o serviço. Para persistência real, considere: - Mudar `DB_PATH` para `/data/oportunidades.db` E adicionar disco persistente no Render (paga $1/mês a mais) - Ou usar PostgreSQL grátis do Render Para o MVP, o `/tmp` é suficiente — se reiniciar, a próxima coleta repõe os dados.
